# 相亲模拟器 - 游戏规格说明书

## 1. 项目概述

- **项目名称**: 相亲模拟器
- **项目类型**: 单页网页游戏 (HTML + CSS + JavaScript)
- **核心功能**: 玩家通过选择对话选项与随机匹配的相亲对象进行互动，根据好感度触发不同结局
- **目标用户**: 年轻网民、喜欢幽默社交游戏的用户

## 2. UI/UX 规格

### 2.1 页面结构

**开始界面**
- 游戏标题（居中，大字体）
- 随机副标题（每次刷新不同）
- 「开始相亲」按钮（主按钮样式）
- 底部：今日相亲人数统计
- 右上角：「历史记录」按钮

**匹配界面**
- 人物卡片（头像、名称、标签）
- 「点击查看对方的自我介绍」按钮
- 弹窗：第一人称详细介绍

**对话界面**
- 顶部：对方头像 + 名称 + 好感度数值
- 中间：聊天记录区域（可滚动）
- 底部：三个选项按钮

**结局界面**
- 结局名称（大字）
- 结局描述
- 「生成分享卡片」按钮
- 「再相一次」按钮

**历史记录界面**
- 列表页：按时间倒序的相亲记录
- 详情页：完整对话历史

### 2.2 视觉设计

**色彩方案**
- 背景色: #1a1a2e (深蓝紫)
- 卡片背景: #16213e (深蓝)
- 主按钮: #e94560 (玫红)
- 次按钮: #0f3460 (深蓝)
- 文字主色: #ffffff
- 文字次色: #a0a0a0
- 好感度正: #4ade80 (绿色)
- 好感度负: #f87171 (红色)
- 选项按钮: #533483 (紫色)

**字体**
- 标题: "ZCOOL KuaiLe", "Ma Shan Zheng", cursive (有趣的中文字体)
- 正文: "Noto Sans SC", sans-serif

**动画**
- 按钮悬停：轻微放大 + 发光
- 好感度变化：数值跳动动画
- 聊天记录：新消息滑入
- 页面切换：淡入淡出

### 2.3 响应式

- 移动端优先设计
- 最大宽度: 480px (居中显示)
- 全屏适配

## 3. 功能规格

### 3.1 随机副标题池
```
- "今天你相了吗？"
- "对方正在输入… 你的葬礼致辞"
- "本游戏已获国家反诈中心友情提示"
- "建议在闺蜜陪同下游玩"
- "每局都会有一个正常人受伤"
- "你的下一个对象，何必是个人"
- "相亲市场的最后一根救命稻草"
```

### 3.2 人设池 (7个角色)

| ID | 名称 | 标签 | 头像emoji |
|---|---|---|---|
| 1 | 杀猪盘金融哥 | 币圈大佬，只差你一个 | 🤑 |
| 2 | 只是一个普普通通的优质青年 | 只是一个普普通通的优质青年 | 😐 |
| 3 | 会疼人大叔 | 年纪大=会疼人 | 👴 |
| 4 | 妈宝工程师 | 我妈说… | 👨‍💻 |
| 5 | 文艺穷青年 | 我的灵魂很高贵 | 🎭 |
| 6 | 猥琐健身哥 | 我练得很壮哦～嘿嘿 | 💪 |
| 7 | 精致利己孔雀男 | 我的生活品味很贵 | 🦚 |

### 3.3 对话系统

- 每角色 10-15 轮对话
- 每轮结构：系统剧情 → 对方发言 → 3个选项
- 好感度变化：正常选项 +5~+15，抽象选项 -5~-20，极端选项 -15~-30

### 3.4 好感度系统

- 初始值: 0
- 范围: -100 到 100
- 实时显示，正数绿色，负数红色

### 3.5 结局触发

| 好感度 | 结局类型 |
|---|---|
| ≥60 | 灾难性后续（高好感） |
| -20 ~ 59 | 普通结局 |
| ≤-21 | 气走对方 |

### 3.6 分享卡片内容

- 游戏标题
- 对方头像 + 名称
- 最终好感度
- 结局名称 + 描述
- 随机吐槽文案

### 3.7 数据存储

- localStorage 存储历史记录
- 记录结构: {时间, 角色ID, 好感度, 结局, 对话历史}
